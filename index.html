Fun With Khazalid¶

For my final project, I would like to explore an imagined language called Khazalid spoken by the dwarves in the Warhammer Fantasy. The imagined universe was created early in 1983 as a tabletop game, and from then on many novels, comics, and video games about Warhammer Fantasy were created and many of them featured dwarves and their language Khazalid. Therefore, although Khazalid is an imagined language, there are many sources available about its grammar, pronunciation, writing, and lexicon.

The project features an alphabet translator, an Khazalid-to-English translator, and a Khazalid text generator.

Alphabet Translation¶

Khazalid has its own writing called Klinkarhun (See image below). Most Khazalid texts online are still using the English alphabet. This alphabet translator aims at providing a mysterious feeling about this imagined language.

On my computer, there is one png image corresponding to each alphabet. The function below takes in a list of image names, and displays the corresponding images.

The code below implements the alphabet translation. It takes in English characters and returns a list of image names, which can then be fed into the function above in order to display the Klinkarhun.

The alphabet translation does not use a FST. It implements a greedy translator. It is greedy in a sense that it will check the longest prefix that matches a rule and then apply that rule. The translator is conceptually simpler than FST and easier to implement.

def transduce_klinkarhun(xs):
    originalxs = "".join(list(xs))
    ys = []
    xs = xs.lower()
    while xs:
        for (k, v) in alphabetmap:
            k = k.lower()
            v = v.lower()
            if xs.startswith(k):
                xs = xs[len(k):]
                y = v
                break
        else:
            print(f'no rules apply \n{xs}\n{originalxs}')
            assert(0)
        ys.append(y)
    return ys

CFG-based Khazalid Generator¶

Khazalid grammar is very similar to the SVO structure of English. Most of words in Khazalid are nouns that represent concrete objects, but there are some rules that lead to some variation (such as a change in part of speech and tense):

  • plural suffix: -i
  • noun-to-verb (present-tense) suffix: -it
  • noun-to-verb (past-tense) suffix: -ed
  • noun-to-verb (future-tense) prefix and suffix pair: an- + noun root + -it
  • noun-to-verb (complete-perfect-tense) prefix and suffix pair: ad + noun root + et
  • noun-to-verb (future-perfect-tense) prefix and suffix pair: anad + noun root + et

Despite all these variations, Khazalid grammar is simpler and results in a smaller CFG, due to a smaller set of prepositions, adverbs, and adjectives and the lack of infinitive and determiner. The code below defines a base grammar which I will procedually add in rules from an external text file.

import random

def gen(g,init):
    if init not in g:
        return [init]
    else:
        output = []
        if len(g[init]) == 0:
            print(init, 'has no productions')
            return []
        rhs = random.choice(g[init]) 
        for i in rhs:
            x = gen(g,i)
            if x != '':
                output.extend(x)
        return output

Khazalid-to-English Translator¶

Now we can generate khazalid, and let us know its meaning. The goal is to, given a khazalid sentence in English alphabet, output its English translation. To do so, I parse the external Khazalid dictionary to get an equivalent English word for each Khazalid word. Since Khazalid grammar is a subset of English grammar, translation can be as simple as word-by-word.

First let us parse the dictionary:

In [137]:
kha2eng = dict()
with open("khazalid.txt", "r") as file:
    lines = file.readlines()
    for line in lines:
        sp = line.split(';')
        if len(line) == 0 or len(sp) < 3: continue 
        word, meaning = sp[1].strip().lower(), sp[2].strip().lower()
        kha2eng[word] = meaning

def translate(kha):
    eng = [kha2eng[k.lower()] for k in kha]
    eng[0] = eng[0].capitalize()
    return eng
